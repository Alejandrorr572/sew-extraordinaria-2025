Documentación del Proyecto

Descripción General

Este proyecto implementa un Sistema de Reservas Turísticas para el concejo de Siero, integrando información turística, actividades culturales, gastronomía, y permitiendo la gestión completa de reservas de recursos turísticos.

Módulos Implementados

1. Mejora de la Interfaz General

  • Eliminación de div, estilos en línea, clases e IDs innecesarios
  • Implementación de HTML semántico y atributos de accesibilidad
  • Separación clara entre contenido (HTML), presentación (CSS) y comportamiento (JavaScript)
  • Adaptación responsive para múltiples dispositivos

2. Sistema de Reservas PHP

  • Arquitectura MVC (Modelo-Vista-Controlador) completa
  • Gestión de usuarios (registro e inicio de sesión)
  • Catálogo de recursos turísticos filtrable
  • Sistema de reservas con confirmación automática
  • Panel de control de usuario con estadísticas
  • Gestión de disponibilidad para agosto y septiembre

3. Optimizaciones Avanzadas

  • Seguridad:
    • Validación de formularios tanto en cliente como en servidor
    • Prevención de inyección SQL mediante consultas parametrizadas
    • Gestión de sesiones seguras
    • Encriptación de contraseñas
  • Optimización de bases de datos:
    • Transacciones seguras para garantizar la integridad de los datos
    • Normalización de tablas para evitar redundancias
    • Índices para mejorar el rendimiento de consultas frecuentes
  • Accesibilidad:
    • Markup semántico en todas las vistas
    • Atributos ARIA donde es necesario
    • Navegación por teclado mejorada
    • Alto contraste en la interfaz

Mejoras Implementadas en el Sistema de Reservas

  1. Modernización general de la interfaz: Eliminación de estilos en línea y uso de HTML semántico en todas las vistas PHP
  2. Corrección de rutas CSS: Todas las referencias a hojas de estilo ahora son correctas y funcionan en todo el sistema
  3. Arreglo del registro de usuarios: Corregido error en el campo de confirmación de contraseña
  4. Navegación mejorada: Integración perfecta entre el sitio principal HTML y el sistema de reservas PHP
  5. Inicialización automática: Creación de recursos por defecto si no existen al iniciar sesión
  6. Simplificación de la interfaz: Eliminación de filtros complejos e imágenes para una experiencia más limpia
  7. Flujo directo de reservas: Nuevo sistema de creación de reservas más intuitivo
  8. Arquitectura modular: Servicios separados para Recursos y Reservas
  9. Gestión transaccional segura: Prevención de problemas con transacciones anidadas
  10. Estadísticas actualizadas: El panel principal y la vista de reservas muestran datos consistentes
  11. Disponibilidad controlada: Nuevas disponibilidades limitadas a agosto/septiembre para mejor planificación

Componentes Técnicos

  • Frontend: HTML5, CSS3, JavaScript nativo, sin dependencias de frameworks
  • Backend: PHP 8 con arquitectura MVC personalizada
  • Base de datos: MySQL con procedimientos almacenados para operaciones críticas
  • APIs: Integración de servicios meteorológicos y manejo de datos XML para rutas
  • Persistencia: Gestión de sesiones, cookies, y almacenamiento local donde corresponde
